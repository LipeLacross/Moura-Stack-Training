🧬dbt

      
Modelos stg_sales (silver) e fct_sales (gold) implementados no projeto.

      
Modelo stg_sales.sql (silver): Normaliza e garante coluna total.

      
select
  order_id,
  region,
  product,
  quantity,
  unit_price,
  coalesce(total, quantity * unit_price) as total
from sales;

      
Modelo fct_sales.sql (gold): Agrega receita e quantidade por produto.

      
select
  product,
  sum(quantity)    as total_quantity,
  sum(total)       as total_revenue,
  avg(unit_price)  as avg_unit_price
from stg_sales
group by product;

      
O dbt organiza dados em camadas: Silver (normalização) e Gold (agregação analítica). Permite análises confiáveis e integrações BI. Saiba mais

    

    

      
🐳Docker

      
Infraestrutura pronta para deploy local ou cloud.

        Dockerfile:
        
FROM python:3.11-slim
WORKDIR /app
ENV PYTHONDONTWRITEBYTECODE=1
ENV PYTHONUNBUFFERED=1
RUN apt-get update && apt-get install -y \
    build-essential \
    openjdk-17-jre-headless \
    && rm -rf /var/lib/apt/lists/*
ENV JAVA_HOME=/usr/lib/jvm/java-17-openjdk-amd64
ENV PATH="${JAVA_HOME}/bin:${PATH}"
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY . .
COPY public ./public
EXPOSE 8000
CMD ["uvicorn", "app.backend.main:app", "--host", "0.0.0.0", "--port", "${PORT:-8000}"]


        docker-compose.yml:
        
version: "3.9"
services:
  api:
    build: .
    container_name: moura_api
    ports:
      - "8000:8000"
    env_file:
      - .env
    volumes:
      - ./data:/app/data
      - ./public:/app/public
      - ./app:/app/app
      - ./sql:/app/sql
    restart: unless-stopped


        Exemplo: docker build -t moura-api . 

        docker run -p 8000:8000 moura-api 

        Compose: docker-compose up --build
